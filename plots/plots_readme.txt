# Plots for the KROEG Index

###################
### What's here ###
###################

This folder holds the pre-rendered map files shown by the website's dashboard (index.qmd).
Every file is generated by scripts/panel_mapping.ipynb from data/KROEG_index.gpkg; nothing in
this folder is hand-edited. There are 55 files in total: 54 maps (9 sectors x 6 views) plus
one combined file.

	<sector>.html
		The national map for that sector: every PC4 area in the Netherlands, coloured by
		its 0-100 ranking on that sector, compared against all other PC4 areas.
		9 files, one per sector:
			third_places_total_ranked
			third_places_non_medical_total_ranked
			third_places_non_medical_non_retail_total_ranked
			third_places_medical_ranked
			third_places_food_beverage_ranked
			third_places_retail_ranked
			third_places_community_ranked
			third_places_cultural_ranked
			third_places_sport_leisure_ranked

	<sector>_by_urbanisation_<level>.html
		The same sector, but showing only PC4 areas at one urbanisation level (1 to 5),
		ranked against each other rather than against the whole country.
		9 sectors x 5 levels = 45 files, e.g.
			third_places_food_beverage_ranked_by_urbanisation_3.html

	dashboard.html
		A single combined Panel template containing all sectors together. This is a
		leftover export from the notebook and is not the file the live website loads;
		the dashboard page loads the individual files above into its iframe instead.

Each file is a self-contained, interactive map (CartoLight basemap, blue-to-orange colour
scale, pan/zoom/hover) with all of its Bokeh/Plotly JavaScript and styling embedded inline, so
every file works on its own when opened directly or loaded into the website's iframe.


########################
### Naming matters ###
########################

The website's dropdown menus build a file name from the chosen sector and urbanisation level
and load it straight from this folder (see index.qmd). The file names here must therefore
match that pattern exactly — renaming or removing a file will break the corresponding option
in the dashboard.


######################
### Regenerating ###
######################

These files are not edited by hand. To update them, re-run scripts/panel_mapping.ipynb, which
overwrites everything in this folder, then rebuild the site with 'quarto render'.
